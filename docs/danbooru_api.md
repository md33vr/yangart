# Danbooru API — понятная шпаргалка

Краткое и практичное описание API сайта Danbooru применительно к этому боту
(`cogs/art_grabber.py`). За основу взята официальная документация:
<https://danbooru.donmai.us/wiki_pages/help:api>

---

## 1. Базовый адрес

| Назначение | URL |
|------------|-----|
| Боевой сервер | `https://danbooru.donmai.us` |
| Тестовый сервер (для отладки) | `https://testbooru.donmai.us` |

Чтобы получить **JSON**, к пути добавляется `.json`. Бот использует список постов:

```
https://danbooru.donmai.us/posts.json
```

Если нужен XML — добавляется `.xml`, но нам нужен именно JSON.

---

## 2. Основные эндпоинты

| Метод и путь | Что делает |
|--------------|------------|
| `GET /posts.json` | Список постов с фильтром по тегам (то, что использует бот) |
| `GET /posts/:id.json` | Один пост по его ID |
| `GET /posts/random.json` | Один случайный пост |
| `GET /tags.json` | Поиск тегов |
| `GET /artists.json` | Список авторов |

Дальше всё про `GET /posts.json`.

---

## 3. Параметры запроса (`GET /posts.json`)

Передаются как обычные query-параметры (`?tags=...&limit=...`).

| Параметр | Тип | Значение |
|----------|-----|----------|
| `tags` | строка | Условие поиска. Теги разделяются **пробелами**, внутри одного тега пробел заменяется на `_`. Несколько тегов работают по логике **И**. |
| `limit` | число | Сколько постов вернуть. Максимум **200** для постов. |
| `page` | число / строка | Пагинация (см. ниже). |

> ⚠️ **Важно про `random`.**
> У `/posts.json` **нет** отдельного булева параметра `random`. Случайность
> задаётся метатегом внутри `tags`: `order:random`.
> То есть правильнее `tags="cat_girl order:random"`, а не `params["random"]=True`.
> (Параметр `random=true` исторически работал, но официально поддерживается
> именно `order:random`, и он надёжнее.)

### Пример запроса, как его собирает бот

```python
params = {
    "tags": "blue_hair order:random rating:g",
    "limit": 1,
}
```

Превращается в:

```
GET /posts.json?tags=blue_hair+order:random+rating:g&limit=1
```

## 4. Синтаксис тегов и метатеги

Внутри строки `tags` можно комбинировать обычные теги и «метатеги» (служебные
фильтры вида `ключ:значение`).

### Логические операции

| Операция | Синтаксис | Пример |
|----------|-----------|--------|
| И (все теги сразу) | пробел | `blonde_hair blue_eyes` |
| НЕ (исключить) | `-` перед тегом | `cat_girl -comic` |
| ИЛИ | `or` или `~` | `cat_girl or dog_girl` |

### Полезные метатеги

| Метатег | Что делает | Примеры значений |
|---------|------------|------------------|
| `rating:` | Возрастной рейтинг | `g`, `s`, `q`, `e` (см. таблицу ниже) |
| `order:` | Сортировка | `random`, `score`, `rank`, `id`, `favcount`, `mpixels` |
| `score:` | Фильтр по рейтингу-оценке | `score:>100`, `score:50..200` |
| `id:` | Фильтр по ID | `id:1000..2000` |
| `age:` | По дате загрузки | `age:..1week` (за последнюю неделю) |
| `width:` / `height:` | Размер картинки | `width:>=1920` |

### Значения `rating:`

| Код | Название | Смысл |
|-----|----------|-------|
| `g` | general | безопасный контент |
| `s` | sensitive | лёгкий/пограничный |
| `q` | questionable | откровенный намёк |
| `e` | explicit | контент 18+ |

Чтобы получать только безопасные арты: добавьте `rating:g` в `tags`.
В коде бота сейчас откидываются `e` и `q` уже после запроса — надёжнее сразу
ограничить выдачу через `rating:g` (или `-rating:e -rating:q`).

> ⚠️ **Лимит на число тегов.** Анонимные пользователи и базовые аккаунты
> ограничены **двумя тегами** в одном поиске (метатеги вроде `order:` и
> `rating:` тоже считаются тегами). Больше тегов — нужен авторизованный
> аккаунт (Gold+).

## 5. Пагинация

| Форма | Значение |
|-------|----------|
| `page=2` | Вторая страница (есть ограничение по максимуму) |
| `page=b123456` | Посты **до** поста с ID 123456 (before) |
| `page=a123456` | Посты **после** поста с ID 123456 (after) |

---

## 6. Аутентификация (необязательно)

Для чтения публичных постов ключ **не нужен**. Но с ключом снимаются часть
ограничений (например, лимит в 2 тега). Ключ генерируется в профиле на сайте.

Способы передачи (любой из):

```python
# 1) Как query-параметры
params["login"] = "username"
params["api_key"] = "ВАШ_КЛЮЧ"

# 2) HTTP Basic Auth
requests.get(url, auth=("username", "ВАШ_КЛЮЧ"))
```

> 🔒 Ключ — это пароль. Не коммитьте его в репозиторий, держите в `.env`
> рядом с `TOKEN` и `GUILD_ID`.

---

## 7. User-Agent

Danbooru **требует** осмысленный заголовок `User-Agent`, чтобы опознавать ботов.
Формат: `ИмяБота/Версия (контакт)`.

```python
headers = {"User-Agent": "YangartBot/1.0 (discord bot by username)"}
```

Текущее значение `"prop.spell"` валидно, но лучше указать что-то описательное.

---

## 8. Лимиты по частоте запросов (rate limit)

- **Чтение:** ~**10 запросов в секунду** (глобально, делится между аккаунтом и IP).
- Превышение → HTTP `429 User Throttled`.
- Актуальные лимиты приходят в заголовке `x-rate-limit` и доступны на `/rate_limits`.

Для одиночной команды Discord это не проблема, но если делать массовые запросы —
ставьте паузы.

---

## 9. Что возвращается (поля поста)

`GET /posts.json` возвращает **массив** объектов-постов. Если ничего не найдено —
**пустой массив** `[]`. Полезные поля одного поста:

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | ID поста |
| `rating` | string | `g` / `s` / `q` / `e` |
| `file_url` | string | Прямая ссылка на оригинал картинки |
| `large_file_url` | string | Увеличенный превью-вариант |
| `preview_file_url` | string | Маленькое превью |
| `file_ext` | string | Расширение (`jpg`, `png`, `gif`, `webm`…) |
| `image_width` / `image_height` | int | Размеры в пикселях |
| `score` | int | Оценка сообщества |
| `tag_string` | string | Все теги поста через пробел |
| `source` | string | Источник (откуда залито) |
| `created_at` | string | Дата загрузки (ISO 8601) |

> ⚠️ **`file_url` может отсутствовать.** Для забаненных или доступных только
> Gold-аккаунтам постов поля `file_url` в ответе не будет. Поэтому используйте
> `post.get("file_url")` и проверяйте на `None`, иначе `post["file_url"]`
> бросит `KeyError`.

### Пример ответа (сокращённо)

```json
[
  {
    "id": 1234567,
    "rating": "g",
    "score": 42,
    "image_width": 1920,
    "image_height": 1080,
    "file_ext": "jpg",
    "tag_string": "1girl blue_hair smile",
    "file_url": "https://cdn.donmai.us/original/ab/cd/abcd...jpg"
  }
]
```

---

## 10. Коды ответов

| Код | Значение |
|-----|----------|
| `200 OK` | Успех |
| `400 Bad Request` | Ошибка в параметрах |
| `401 Unauthorized` | Неверная аутентификация |
| `403 Forbidden` | Доступ запрещён |
| `404 Not Found` | Не найдено |
| `422` / `424` | Неверные данные/параметры |
| `429 Too Many Requests` | Превышен лимит запросов |

В Python `response.raise_for_status()` сам выбросит исключение на коды 4xx/5xx.

## 11. Минимальный рабочий пример

```python
import requests

BASE_URL = "https://danbooru.donmai.us/posts.json"

def get_random_art(user_tags: str) -> str | None:
    # пользователь вводит теги через пробел: "blue hair smile"
    # внутри тега пробел -> '_', между тегами оставляем пробел
    tags = " ".join(t.replace(" ", "_") for t in user_tags.split())

    params = {
        "tags": f"{tags} order:random rating:g",  # сразу только безопасные
        "limit": 1,
    }
    headers = {"User-Agent": "YangartBot/1.0 (discord bot)"}

    with requests.Session() as session:
        session.headers.update(headers)
        response = session.get(BASE_URL, params=params, timeout=10)
        response.raise_for_status()
        data = response.json()

    if not data:                      # пустой массив = ничего не найдено
        return None

    return data[0].get("file_url")    # .get(), т.к. поля может не быть
```

## 12. Замечания к текущему коду бота

Не относится к самой документации API, но пригодится при правке
`cogs/art_grabber.py`:

1. **`tags` должен быть строкой, а не списком.** Сейчас
   `re.sub(r' ', '_', tags).split(":")` даёт список, и `requests`
   сериализует его не так, как ожидает API. Соберите одну строку с тегами
   через пробел.
2. **`random=True` → `order:random` внутри `tags`** (см. раздел 3).
3. **Фильтр рейтинга лучше делать в запросе** (`rating:g`), а не циклом после.
   Текущий `while RATING in ("e", "q")` к тому же зацикливается логически:
   `n` увеличивается, но всегда берётся `data[1]`, а `RATING` не пересчитывается.
4. **После `if not data:` нужен `return`** — иначе код пойдёт дальше и упадёт
   на `data[0]`.
5. **Берите `post.get("file_url")`** и проверяйте на `None`.
